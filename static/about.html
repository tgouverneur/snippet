About

Secure Snippet is an open-source software which allow text data to be transmitted between two people in a (relatively) secure way.

How does it work?

When the creator of the snippet fill in the form, the content of the snippet is sent to the server through an SSL enabled connection. The server will then generate a 32 bytes key and encrypt the content with AES256 cipher and store the content in the database. The key, will be returned to the creator as part of a formed URL that can be given to the destination of the snippet.

When the destinator of the snippet points to that URL, the content is fetched from the database, decrypted and sent back to his browser. The content is removed from the database at that same moment.

Why is it (relatively) secure?

Form the time the creator of the snippet send the content to the moment the destinator reads it, the content is either transmitted over an encrypted channel or is itself, encrypted. The key is never stored on the server side, rendering the process of recovering the content without knowledge of the URL of the snippet, impossible. There is no logs containing the key or the content of the snippet.

In most companies, users tend to send passwords over email, slack, chat, etc.. This process exposes sensitive information in public with no protection. This secure snippet comes to address this problem by making it slightly more secure to exchange information while keeping the whole process easy and simple!

Why is it (relatively) insecure?

The entire snippet system is relying on the browsers of both creator and destinator. This itself can be considered a security risk if the content of the snippet is really sensitive. To transmit ultimately sensitive content, you may consider using other way of encryption like, for example, PGP.
